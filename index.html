FinLocker Analytics

Welcome to the FinLocker Analytics documentation.

Overview

This project contains analytics services and applications for the FinLocker platform, providing comprehensive data processing, reporting, and user engagement tracking capabilities.

Key Components

Core Libraries

  • FinLocker.Analytics.Application: Core application layer with business logic and services
  • FinLocker.Analytics.Contracts: Contracts and interfaces for queue messaging
  • FinLocker.Analytics.Database: Database layer with Entity Framework contexts and models
  • FinLocker.Analytics.Types: Shared types, enums, and data models

Services

  • FinLocker.Analytics.Internal.Api: Internal REST API service
  • FinLocker.Analytics.Queue: Background queue processing worker

Background Jobs

  • LoanOfficerReportJob: Generates comprehensive loan officer performance reports
  • SupportingDataJob: Processes and synchronizes supporting data
  • SyncJob: Handles data synchronization operations
  • UserAttributesJob: Processes user attribute data and analytics

Key Features

📊 Analytics & Reporting

  • User engagement tracking
  • Loan officer performance metrics
  • Application usage statistics
  • Email campaign analytics

🔄 Data Processing

  • Real-time data synchronization
  • Parquet file generation for analytics
  • Background job processing
  • Queue-based message handling

🗄️ Data Sources

  • FinLocker user data
  • Company information
  • Prospect tracking
  • User attributes and behaviors

Development

Prerequisites

  • .NET 8.0 SDK
  • SQL Server or compatible database
  • Azure Service Bus (for queue processing)

Key Technologies

  • Entity Framework Core: Database access and migrations
  • Parquet.Net: Analytics file generation
  • Azure Service Bus: Message queuing
  • ASP.NET Core: Web API framework
