The FinLocker GenAI system includes several background processing components that handle AI content generation, evaluation, and reporting tasks.

Available Jobs and Queue Workers

AI Evaluator Queue

Background worker that processes AI evaluation requests, scoring generated content for quality, relevance, and accuracy. Handles feedback processing and model performance assessment.

AI Runner Queue

Background worker responsible for executing AI content generation requests. Processes queue messages to generate intelligent responses, documents, and other AI-powered content.

Market Report Sync Job

Automated job that synchronizes market data and generates AI-powered market reports. Handles data processing, analysis, and report generation using AI models.

Queue Processing Architecture

All queue workers are designed to process messages asynchronously and can be deployed as:

  • Azure Container Instances
  • Kubernetes pods
  • Azure Functions (consumption plan)
  • Background services in Azure App Service

AI Processing Flow

  1. Request Reception: API services receive AI requests and queue them
  2. Queue Processing: Background workers pick up and process requests
  3. AI Generation: Content is generated using AI models
  4. Evaluation: Generated content is optionally evaluated for quality
  5. Storage: Results are stored in Cosmos DB
  6. Notification: Completion status is communicated back to requesters

For detailed implementation and configuration, refer to each component's specific documentation.
