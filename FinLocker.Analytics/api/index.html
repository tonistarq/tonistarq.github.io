API Documentation

This section contains the complete API reference for all FinLocker Analytics components.

Overview

The FinLocker Analytics solution is organized into several key projects, each serving a specific purpose in the analytics pipeline. Browse the sections below to explore the APIs for each component.

Core Libraries

FinLocker.Analytics.Application

The main application layer containing business logic, services, and application orchestration.

Key Features:

  • Service implementations for data processing
  • Application extensions and configuration
  • Validation logic and utilities
  • Integration with external services

FinLocker.Analytics.Database

Database access layer with Entity Framework contexts and data models.

Key Features:

  • FinLockerDbContext for main application data
  • SynapseDbContext for analytics data warehouse
  • Entity models and relationships
  • Database configuration and options

FinLocker.Analytics.Types

Shared types, enums, and data models used across all projects.

Key Features:

  • Common enums and constants
  • Data transfer objects (DTOs)
  • Shared model classes
  • Type extensions and utilities

FinLocker.Analytics.Contracts

Contracts and interfaces for queue messaging and service communication.

Key Features:

  • Queue message contracts
  • Service interfaces
  • Event definitions
  • Communication protocols

Services

FinLocker.Analytics.Internal.Api

Internal REST API service for analytics operations.

Key Features:

  • RESTful endpoints for internal use
  • Authentication and authorization
  • Request/response models
  • API configuration and middleware

FinLocker.Analytics.Queue

Background queue processing worker service.

Key Features:

  • Queue message processing
  • Background worker implementation
  • Queue configuration and setup
  • Error handling and retry logic

Background Jobs

FinLocker.Analytics.LoanOfficerReportJob

Console application for generating loan officer performance reports.

Key Features:

  • Report generation logic
  • Data aggregation and analysis
  • Export capabilities
  • Scheduling and automation support

FinLocker.Analytics.SupportingDataJob

Console application for processing supporting data required for analytics.

Key Features:

  • Data validation and cleansing
  • Reference data updates
  • Data transformation pipelines
  • Integration with external data sources

FinLocker.Analytics.SyncJob

Console application handling data synchronization between systems.

Key Features:

  • Multi-system data synchronization
  • Conflict resolution strategies
  • Incremental sync capabilities
  • Data consistency validation

FinLocker.Analytics.UserAttributesJob

Console application for processing user attributes and behavioral analytics.

Key Features:

  • User behavior analysis
  • Attribute calculation and updates
  • Engagement metrics processing
  • User segmentation logic

Common Patterns

Configuration

All projects follow consistent configuration patterns using:

  • appsettings.json for environment-specific settings
  • Options pattern for strongly-typed configuration
  • Environment variable overrides

Dependency Injection

Standard .NET dependency injection is used throughout:

  • Service registration in Program.cs
  • Interface-based service design
  • Scoped lifetime management

Logging

Structured logging using:

  • ILogger<T> injection
  • Consistent log levels and messages
  • Performance and diagnostic logging

Error Handling

Consistent error handling approach:

  • Global exception handlers
  • Custom exception types
  • Structured error responses
